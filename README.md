# 🍽️ Система бронирования столиков в ресторанах (Вариант 31)

Это курсовой проект по дисциплине «Методы и технологии программирования». Система представляет собой веб-приложение для управления бронированием столиков в ресторанах.

## 🚀 Быстрый запуск
Проект настроен для запуска «в один клик» с помощью Docker.

**Предпосылки:** Установленный [Docker Desktop](https://www.docker.com/products/docker-desktop/).

1. Склонируйте этот репозиторий или скачайте архив с кодом.
2. Откройте терминал в корне проекта.
3. Запустите команду:
   ```bash
   docker-compose up -d
   ```
4. **Готово!** Приложение доступно по адресу: [http://localhost:8000/](http://localhost:8000/)

---

## 🛠 Функциональные возможности
- **Визуальный интерфейс:** Интерактивная карта залов ресторанов.
- **Умное бронирование:** Система не позволяет забронировать уже занятый стол.
- **Автоматизация:** При первом запуске база данных автоматически наполняется тестовыми данными (2 ресторана, столы с разными статусами), чтобы приложение не было пустым.
- **Асинхронность:** Бронирование происходит мгновенно без перезагрузки страницы (через JavaScript Fetch API).

## 📐 Технический стек и архитектура
- **Backend:** Python 3.11, FastAPI (высокопроизводительный асинхронный фреймворк).
- **Database:** PostgreSQL (реляционная БД), SQLAlchemy 2.0 (ORM).
- **Validation:** Pydantic V2 (строгая типизация и валидация данных).
- **Frontend:** Jinja2 Templates, HTML5, CSS3 (Bootstrap 5 CDN).
- **Infrastructure:** Docker, Docker Compose.

### Схема базы данных:
- `restaurants` (Рестораны) $\rightarrow$ Один-ко-многим $\rightarrow$ `tables` (Столы)
- `tables` (Столы) $\rightarrow$ Один-ко-многим $\rightarrow$ `bookings` (Бронирования)

Все эндпоинты задокументированы автоматически с помощью Swagger UI.
После запуска проекта перейдите по ссылке:
👉 [http://localhost:8000/docs](http://localhost:8000/docs)

## 🧪 Тестирование и качество кода
- **Модульные тесты:** Реализовано тестирование основных эндпоинтов и бизнес-логики бронирования с использованием `pytest` и `httpx`.
- **Запуск тестов в Docker:**
  ```bash
   docker-compose exec web pytest
  ```
- **SAST/Линтинг:** Проект соответствует стандартам чистоты кода. Настроен линтер **Ruff** (конфигурация в `pyproject.toml`), который проверяет код на наличие ошибок, багов и соблюдение стиля.

## 📂 Структура проекта
- `app/` — основной код приложения.
  - `models.py` — модели SQLAlchemy.
  - `schemas.py` — схемы Pydantic.
  - `crud.py` — бизнес-логика.
  - `main.py` — точка входа и маршруты.
  - `templates/` — HTML-шаблоны.
- `tests/` — интеграционные тесты.
- `docker-compose.yml` — конфигурация инфраструктуры.
